01

Formatting Rules

Quy tắc định dạng
1.1

Indentation — Thụt Lề

Sử dụng 4 spaces cho mỗi level

error
javascript
// ❌ SAI — 2 spaces
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// ✅ ĐÚNG — 4 spaces
function calculateTotal(items) {
    return items.reduce((sum, item) => sum + item.price, 0);
}

switch (type) {
    case "A":
        return "Type A";
    default:
        return "Unknown";
}
1.2

Line Endings — Kết Thúc Dòng

Dùng LF (Unix), không dùng CRLF (Windows)

error
.gitattributes
// Config Git: core.autocrlf = false
* text=auto eol=lf
1.3

Quotes — Dấu Ngoặc Kép

Dùng double quotes, cho phép template literals

error
javascript
const name = 'John';           // ❌ single quotes
const msg  = "Hello " + name;  // ❌ concatenation

const name = "John";           // ✅
const msg  = `Hello ${name}`;  // ✅ template literal
1.4

Semicolons — Dấu Chấm Phẩy

Bắt buộc có ; ở cuối mỗi statement

error
javascript
const x = 1   // ❌ thiếu semicolon
const y = 2

const x = 1;  // ✅
const y = 2;  // ✅
1.5

Trailing Spaces

Không cho phép khoảng trắng ở cuối dòng

error
javascript
const x = 1;    ← trailing spaces
const x = 1;   // ✅ clean
1.6

End of File

Bắt buộc có dòng trống ở cuối file

error
javascript
export default function foo() {
    return "bar";
}
↑ dòng trống bắt buộc sau dấu }
1.7

Empty Lines — Dòng Trống

Tối đa 1 dòng trống liên tiếp

error
javascript
function foo() { return 1; }


function bar() { return 2; }  // ❌ 2 dòng trống

function foo() { return 1; }

function bar() { return 2; }  // ✅ 1 dòng trống
1.8

Function Spacing

Khoảng trắng trước () theo loại function

error
javascript
const bar = function() {}   // ❌
const baz = async() => {}   // ❌

function foo() {}           // ✅ named — không có space
const bar = function () {}  // ✅ anonymous — có space
const baz = async () => {}  // ✅ async arrow — có space
1.9

Object & Array Spacing

Object có space trong {}, Array không có space trong []

error
javascript
const obj = {a: 1, b: 2};   // ❌ thiếu space
const arr = [ 1, 2, 3 ];    // ❌ thừa space

const obj = { a: 1, b: 2 }; // ✅
const arr = [1, 2, 3];       // ✅
1.10

Comma Dangle — Dấu Phẩy Thừa

Không cho phép dấu phẩy cuối

error
javascript
const obj = { a: 1, b: 2, };  // ❌
const arr = [1, 2, 3, ];      // ❌

const obj = { a: 1, b: 2 };   // ✅
const arr = [1, 2, 3];        // ✅
1.11

Key Spacing

Không có space trước :, có space sau :

error
javascript
const obj = {a :1, b:2};   // ❌
const obj = { a: 1, b: 2 }; // ✅
1.12

Keyword Spacing

Space trước và sau if, for, while...

error
javascript
if(condition) {}         // ❌
if (condition) {}        // ✅
for (let i = 0; i < 10; i++) {}
1.13

Operator Spacing

Space quanh +, -, =, ===...

error
javascript
const x=1+2;    // ❌
const x = 1 + 2; // ✅
if (x === 1) {}  // ✅ dùng ===, không ==
1.14

Brace Style — 1TBS

Dấu { phải cùng dòng với statement

error
javascript
if (condition)
{                   // ❌ Allman style
    return true;
}

if (condition) {     // ✅ 1TBS
    return true;
}
if (condition) { return true; }  // ✅ single line OK
02

Code Quality Rules

Quy tắc chất lượng code
2.1

Variables — Biến

Cấm var · dùng const ưu tiên · prefix _ cho unused

error
javascript
var x = 1;              // ❌ cấm var
let y = 2;              // ❌ không thay đổi → dùng const

const x = 1;            // ✅
const y = 2;            // ✅
const _unused = 3;      // ✅ bỏ qua warning

let counter = 0;        // ✅ let khi cần thay đổi
counter++;
2.2

Arrow Functions

Ưu tiên arrow function · chỉ dùng () khi nhiều hơn 1 param

error
javascript
arr.map(function(x) { return x * 2; });  // ❌
arr.map((x)=>x*2);                       // ❌

arr.map(x => x * 2);          // ✅ 1 param: không ()
arr.map((x, y) => x + y);    // ✅ 2 params: có ()
2.3

Curly Braces — Ngoặc Nhọn

Bắt buộc {} cho mọi if/else/for/while

error
javascript
if (x) return;                        // ❌
if (x) { return; }                    // ✅
for (let i = 0; i < 10; i++) { console.log(i); }
2.4

Dot Notation

Ưu tiên dot thay bracket khi có thể

error
javascript
obj["property"];         // ❌
obj.property;            // ✅
obj["property-name"];    // ✅ khi key có ký tự đặc biệt
obj[dynamicKey];         // ✅ khi key động
2.5

Equality — Strict Equals

Bắt buộc === và !==

error
javascript
if (x == 1) {}     // ❌
if (x != null) {}  // ❌
if (x === 1) {}    // ✅
if (x !== null) {} // ✅
2.6

Else Return

Bỏ else sau return

warning
javascript
// ⚠️ CẢNH BÁO
if (x) {
    return 1;
} else {
    return 2;
}

// ✅ ĐÚNG
if (x) {
    return 1;
}
return 2;
2.7

Implicit Coercion — Ép Kiểu Ngầm

Dùng explicit conversion

warning
javascript
const x = +"123";     // ⚠️
const y = "" + 123;   // ⚠️

const x = Number("123"); // ✅
const y = String(123);   // ✅
2.8

Return Await

Không cần return await trong async function

error
javascript
async function foo() {
    return await bar(); // ❌
}

async function foo() {
    return bar();       // ✅ await tự động
}
2.9

Promise Reject

Bắt buộc reject với Error object

error
javascript
Promise.reject("error string");         // ❌
Promise.reject(new Error("message"));   // ✅
2.10

Max Line Length

Cảnh báo dòng dài hơn 120 ký tự

warning
javascript
const result = someVeryLongFunctionCall(
    param1,
    param2,
    param3       // ✅ tách dòng khi > 120 chars
);
2.11

Max Depth — Độ Sâu Lồng

Cảnh báo lồng nhau quá 4 levels — dùng early return

warning
javascript
// ✅ Early return pattern
if (!a) { return; }
if (!b) { return; }
if (!c) { return; }
// code chính
2.12

Template Literals

Ưu tiên template literal, không có space trong ${}

warning
javascript
const msg = "Hello " + name + "!"; // ❌
const msg = `Hello ${name}!`;       // ✅ không phải ${ name }
2.13

Duplicate Imports

Cấm import cùng module nhiều lần

error
javascript
import { a } from "mod"; // ❌
import { b } from "mod";

import { a, b } from "mod"; // ✅
03

JSDoc Rules

Quy tắc documentation
3.1

Required JSDoc

Bắt buộc cho functions, methods, classes — không áp dụng với arrow/expression

error
javascript
function calculateTotal(items) {  // ❌ thiếu JSDoc
    return items.reduce((sum, item) => sum + item.price, 0);
}

/**
 * Tính tổng giá trị các items
 * @param {Object[]} items - Danh sách items
 * @returns {number} Tổng giá trị
 */
function calculateTotal(items) {  // ✅
    return items.reduce((sum, item) => sum + item.price, 0);
}
3.2–3.3

JSDoc Format & Required Tags

@param với type+name+desc · @returns nếu có return value

error
javascript
/** @param {string} name */         // ❌ thiếu description
/** Mô tả */ function foo() { return 1; }  // ❌ thiếu @returns

/**
 * Tính tổng giá trị
 * @param {string} name - Tên người dùng
 * @param {number} age  - Tuổi
 * @returns {string} Thông tin người dùng
 */
function getUserInfo(name, age) {
    return `${name} - ${age}`;
}
3.4

Property Documentation

Bắt buộc @property cho mỗi property trong object

error
javascript
/**
 * @property {string} name - Tên người dùng
 * @property {number} age  - Tuổi
 */
const obj = { name: "John", age: 30 };
04

Import Rules

Quy tắc import modules
4.1

Import Order — Thứ Tự Import

Sắp xếp theo nhóm, alphabetize trong từng nhóm

error
1builtin
2external
3internal
4parent ../
5sibling ./
6index
javascript
import "fs";              // ✅ builtin
import "lodash";          // ✅ external
import "../utils/helper"; // ✅ parent
import "./local";         // ✅ sibling
4.2–4.3

No Unresolved & No Cycle

Cấm import không tồn tại · cảnh báo circular dependency

error
javascript
// ✅ Tách shared code ra module riêng để tránh circular
// shared.js
export function sharedFunction() {}

// fileA.js — import từ shared, không từ fileB
import { sharedFunction } from "./shared";
05

Security Rules

Quy tắc bảo mật
5.1

Object Injection

Validate input trước khi dùng làm object key

warning
javascript
obj[userInput];  // ⚠️ nguy hiểm

const allowedKeys = ["key1", "key2"];
if (allowedKeys.includes(userInput)) {
    return obj[userInput]; // ✅
}
5.2

File Path

Sanitize path từ user input

warning
javascript
fs.readFile(userInput);  // ⚠️

const path = require("path");
const safe = path.join(baseDir, path.basename(userInput));
fs.readFile(safe); // ✅
5.3

Eval — CỰC KỲ NGUY HIỂM

Cấm tuyệt đối eval() với expression động

error
javascript
eval(userInput);           // ❌ RCE vulnerability!
const data = JSON.parse(userInput); // ✅
5.4

Timing Attacks

Dùng crypto.timingSafeEqual() cho string comparison bảo mật

warning
javascript
if (password === userPassword) {}  // ⚠️ timing attack

const crypto = require("crypto");
const a = Buffer.from(password);
const b = Buffer.from(userPassword);
if (crypto.timingSafeEqual(a, b)) {} // ✅
06

Promise Rules

Quy tắc xử lý Promise & async
6.1

Always Return

Promise handler phải có return

warning
javascript
promise.then(() => {
    doSomething(); // ⚠️ thiếu return
});

promise.then(() => {
    return doSomething(); // ✅
});
6.2

Catch or Return

Cấm Promise không có catch handler

error
javascript
promise.then(() => {}); // ❌ lỗi bị nuốt

promise.then(() => {}).catch(err => {
    console.error(err); // ✅
});
6.3–6.5

Prefer Async/Await

Ưu tiên async/await thay .then() và callback

warning
javascript
promise.then(result => { console.log(result); }); // ⚠️

fs.readFile(path, (err, data) => { // ⚠️ callback
    if (err) { throw err; }
    console.log(data);
});

const result = await promise;
console.log(result);             // ✅

const data = await fs.promises.readFile(path);
console.log(data);               // ✅

Naming Conventions

Quy tắc đặt tên
Loại Convention Ví dụ
Variables & Functions camelCase userName, calculateTotal()
Constants UPPER_SNAKE_CASE API_BASE_URL, MAX_RETRY_COUNT
Classes PascalCase UserService, OrderController
Private Members _prefix this._name, _privateMethod()
Booleans is/has/should/can isLoading, hasPermission, canEdit
08

File Structure

Cấu trúc và tổ chức file
8.1

File Organization — Thứ Tự Code

Imports → Constants → Types → Functions → Exports

javascript
// 1. Imports (external → internal)
import "fs";
import "lodash";
import "../utils/helper";

// 2. Constants
const API_BASE_URL = "https://api.example.com";

// 3. Functions
/**
 * @param {number[]} numbers
 * @returns {number}
 */
function calculateSum(numbers) {
    return numbers.reduce((sum, n) => sum + n, 0);
}

// 4. Exports
export default calculateSum;
8.2

JSON Files

Indent 2 spaces (chuẩn JSON)

json
{
  "name": "project",
  "version": "1.0.0",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
8.3

Test Files

Rules lỏng hơn: không bắt buộc JSDoc, cho phép callback lồng nhiều

relaxed
javascript
// ✅ Test files không cần JSDoc
describe("calculateSum", () => {
    it("should return sum of numbers", () => {
        expect(calculateSum([1, 2, 3])).toBe(6);
    });
});
✓

Pre-commit Checklist

Kiểm tra trước khi commit
✓ Đảm bảo tất cả trước khi commit
Code pass ESLint (không có lỗi)
Code pass Prettier (format đúng)
Có JSDoc cho tất cả functions / classes / methods
Không có unused variables (hoặc prefix _)
Không có var — chỉ dùng let / const
Dùng === thay ==
Ưu tiên const khi biến không thay đổi
Có error handling cho tất cả Promise (.catch() hoặc try/catch)
Không có security vulnerabilities (object injection, eval, file path)
Import được sắp xếp đúng thứ tự: builtin → external → internal → parent → sibling
File kết thúc bằng newline (end of file)
