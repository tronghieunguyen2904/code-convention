Code Convention - Quy Tắc Viết Code

Tài liệu này mô tả các quy tắc viết code được áp dụng trong dự án, dựa trên cấu hình ESLint.

📋 Mục Lục

1. Formatting Rules - Quy Tắc Định Dạng

1.1. Indentation (Thụt Lề)

Quy tắc: Sử dụng 4 spaces cho mỗi level indentation.

// ❌ SAI
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// ✅ ĐÚNG
function calculateTotal(items) {
    return items.reduce((sum, item) => sum + item.price, 0);
}

// Switch case thụt thêm 1 level
switch (type) {
    case "A":
        return "Type A";
    case "B":
        return "Type B";
    default:
        return "Unknown";
}

1.2. Line Endings (Kết Thúc Dòng)

Quy tắc: Sử dụng LF (Unix) thay vì CRLF (Windows).

// ✅ ĐÚNG: File có LF line endings
// Cấu hình Git: core.autocrlf = false (hoặc true với .gitattributes)

1.3. Quotes (Dấu Ngoặc Kép)

Quy tắc: Sử dụng double quotes cho strings, cho phép template literals.

// ❌ SAI
const name = 'John';
const message = "Hello " + name;

// ✅ ĐÚNG
const name = "John";
const message = `Hello ${name}`;

1.4. Semicolons (Dấu Chấm Phẩy)

Quy tắc: Bắt buộc có dấu chấm phẩy ở cuối mỗi statement.

// ❌ SAI
const x = 1
const y = 2

// ✅ ĐÚNG
const x = 1;
const y = 2;

1.5. Trailing Spaces (Khoảng Trắng Cuối Dòng)

Quy tắc: Không cho phép khoảng trắng ở cuối dòng.

// ❌ SAI
const x = 1;    

// ✅ ĐÚNG
const x = 1;

1.6. End of File (Kết Thúc File)

Quy tắc: Bắt buộc có dòng trống ở cuối file.

// ✅ ĐÚNG: File kết thúc bằng newline
export default function foo() {
    return "bar";
}
// ← Dòng trống này bắt buộc

1.7. Empty Lines (Dòng Trống)

Quy tắc: Tối đa 1 dòng trống liên tiếp.

// ❌ SAI
function foo() {
    return 1;
}


function bar() {
    return 2;
}

// ✅ ĐÚNG
function foo() {
    return 1;
}

function bar() {
    return 2;
}

1.8. Function Spacing (Khoảng Trắng Trước Function)

Quy tắc:

  • Anonymous function: có space trước ()
  • Named function: không có space trước ()
  • Async arrow: có space trước ()
// ❌ SAI
function foo() {}
const bar = function() {}
const baz = async() => {}

// ✅ ĐÚNG
function foo() {}
const bar = function () {}
const baz = async () => {}

1.9. Object & Array Spacing (Khoảng Trắng Trong Object/Array)

Quy tắc:

  • Object: có space bên trong {}
  • Array: không có space bên trong []
// ❌ SAI
const obj = {a: 1, b: 2};
const arr = [ 1, 2, 3 ];

// ✅ ĐÚNG
const obj = { a: 1, b: 2 };
const arr = [1, 2, 3];

1.10. Comma Dangle (Dấu Phẩy Thừa)

Quy tắc: Không cho phép dấu phẩy thừa ở cuối object/array.

// ❌ SAI
const obj = { a: 1, b: 2, };
const arr = [1, 2, 3, ];

// ✅ ĐÚNG
const obj = { a: 1, b: 2 };
const arr = [1, 2, 3];

1.11. Key Spacing (Khoảng Trắng Quanh Dấu Hai Chấm)

Quy tắc: Không có space trước :, có space sau :.

// ❌ SAI
const obj = {a :1, b:2};

// ✅ ĐÚNG
const obj = { a: 1, b: 2 };

1.12. Keyword Spacing (Khoảng Trắng Quanh Keywords)

Quy tắc: Có space trước và sau keywords (if, for, while, etc.).

// ❌ SAI
if(condition) {}
for(let i=0; i<10; i++) {}

// ✅ ĐÚNG
if (condition) {}
for (let i = 0; i < 10; i++) {}

1.13. Operator Spacing (Khoảng Trắng Quanh Operators)

Quy tắc: Có space quanh operators (+, -, =, ==, etc.).

// ❌ SAI
const x=1+2;
if (x==1) {}

// ✅ ĐÚNG
const x = 1 + 2;
if (x === 1) {}

1.14. Brace Style (Phong Cách Dấu Ngoặc Nhọn)

Quy tắc: Sử dụng 1TBS (One True Brace Style) - dấu { cùng dòng.

// ❌ SAI (Allman style)
if (condition)
{
    return true;
}

// ✅ ĐÚNG (1TBS)
if (condition) {
    return true;
}

// ✅ ĐÚNG (single line được phép)
if (condition) { return true; }

2. Code Quality Rules - Quy Tắc Chất Lượng Code

2.1. Variables (Biến)

Quy tắc:

  • Cấm dùng var, bắt buộc dùng let hoặc const
  • Ưu tiên const nếu biến không được gán lại
  • Biến không sử dụng: prefix _ để bỏ qua warning
// ❌ SAI
var x = 1;
let y = 2; // không thay đổi sau đó
const unused = 3;

// ✅ ĐÚNG
const x = 1;
const y = 2;
const _unused = 3; // bỏ qua warning

// ✅ ĐÚNG: let khi cần thay đổi
let counter = 0;
counter++;

2.2. Functions (Hàm)

Quy tắc:

  • Ưu tiên arrow function thay function expression
  • Arrow function: space trước và sau =>
  • Chỉ dùng () khi có nhiều hơn 1 parameter
// ❌ SAI
arr.map(function(x) { return x * 2; });
arr.map((x)=>x*2);
arr.map((x) => x * 2);

// ✅ ĐÚNG
arr.map(x => x * 2);
arr.map((x, y) => x + y);

2.3. Curly Braces (Dấu Ngoặc Nhọn)

Quy tắc: Bắt buộc dùng {} cho tất cả if/else/for/while.

// ❌ SAI
if (x) return;
for (let i = 0; i < 10; i++) console.log(i);

// ✅ ĐÚNG
if (x) { return; }
for (let i = 0; i < 10; i++) { console.log(i); }

2.4. Dot Notation (Ký Hiệu Chấm)

Quy tắc: Ưu tiên dot notation thay bracket notation khi có thể.

// ❌ SAI
obj["property"];

// ✅ ĐÚNG
obj.property;

// ✅ ĐÚNG: bracket khi key có ký tự đặc biệt
obj["property-name"];
obj[dynamicKey];

2.5. Equality (So Sánh)

Quy tắc: Bắt buộc dùng === và !== thay == và !=.

// ❌ SAI
if (x == 1) {}
if (x != null) {}

// ✅ ĐÚNG
if (x === 1) {}
if (x !== null) {}

2.6. Else Return (Else Sau Return)

Quy tắc: Khuyến khích bỏ else sau return.

// ⚠️ CẢNH BÁO
if (x) {
    return 1;
} else {
    return 2;
}

// ✅ ĐÚNG
if (x) {
    return 1;
}
return 2;

2.7. Implicit Coercion (Ép Kiểu Ngầm)

Quy tắc: Tránh implicit type coercion.

// ⚠️ CẢNH BÁO
const x = +"123";
const y = "" + 123;

// ✅ ĐÚNG
const x = Number("123");
const y = String(123);

2.8. Return Await (Return Await)

Quy tắc: Không cần return await trong async function.

// ❌ SAI
async function foo() {
    return await bar();
}

// ✅ ĐÚNG
async function foo() {
    return bar(); // await tự động
}

2.9. Promise Reject (Reject Promise)

Quy tắc: Bắt buộc reject Promise với Error object.

// ❌ SAI
Promise.reject("error string");

// ✅ ĐÚNG
Promise.reject(new Error("error message"));

2.10. Max Line Length (Độ Dài Dòng)

Quy tắc: Cảnh báo dòng code dài hơn 120 ký tự.

// ⚠️ CẢNH BÁO
const veryLongVariableName = someVeryLongFunctionCallWithManyParameters();

// ✅ ĐÚNG: Tách thành nhiều dòng
const veryLongVariableName = someVeryLongFunctionCall(
    param1,
    param2,
    param3
);

2.11. Max Depth (Độ Sâu Lồng Nhau)

Quy tắc: Cảnh báo code lồng nhau quá 4 levels.

// ⚠️ CẢNH BÁO
if (a) {
    if (b) {
        if (c) {
            if (d) {
                if (e) { // quá 4 levels
                    // code
                }
            }
        }
    }
}

// ✅ ĐÚNG: Tách thành nhiều function hoặc dùng early return
if (!a) { return; }
if (!b) { return; }
if (!c) { return; }
if (!d) { return; }
// code

2.12. Template Literals (Template String)

Quy tắc: Ưu tiên template literal thay string concatenation.

// ⚠️ CẢNH BÁO
const msg = "Hello " + name + "!";

// ✅ ĐÚNG
const msg = `Hello ${name}!`;

// ✅ ĐÚNG: Không có space trong ${}
const msg = `Hello ${name}!`; // không phải ${ name }

2.13. Duplicate Imports (Import Trùng Lặp)

Quy tắc: Cấm import cùng module nhiều lần.

// ❌ SAI
import { a } from "mod";
import { b } from "mod";

// ✅ ĐÚNG
import { a, b } from "mod";

3. JSDoc Rules - Quy Tắc Documentation

3.1. Required JSDoc (JSDoc Bắt Buộc)

Quy tắc: Bắt buộc có JSDoc cho:

  • Function declarations
  • Method definitions
  • Class declarations

Không áp dụng cho:

  • Arrow functions
  • Function expressions
// ❌ SAI
function calculateTotal(items) {
    return items.reduce((sum, item) => sum + item.price, 0);
}

// ✅ ĐÚNG
/**
 * Tính tổng giá trị các items
 * @param {Object[]} items - Danh sách items
 * @returns {number} Tổng giá trị
 */
function calculateTotal(items) {
    return items.reduce((sum, item) => sum + item.price, 0);
}

3.2. JSDoc Format (Định Dạng JSDoc)

Quy tắc:

  • Dấu * ở đầu mỗi dòng
  • Alignment của tags
  • Syntax hợp lệ
// ❌ SAI
/** @param {string} name Mô tả */
/** @param {string name */ (thiếu dấu đóng ngoặc)
/** ** @param {string} name */ (nhiều dấu *)

// ✅ ĐÚNG
/**
 * Tính tổng giá trị
 * @param {string} name - Tên người dùng
 * @param {number} age - Tuổi
 * @returns {string} Thông tin người dùng
 */
function getUserInfo(name, age) {
    return `${name} - ${age}`;
}

3.3. Required Tags (Tags Bắt Buộc)

Quy tắc: Bắt buộc có:

  • @param cho mỗi parameter (có type, name, description)
  • @returns nếu function có return (có type, description)
  • Description cho function/class/method
// ❌ SAI
/** @param {string} */ function foo(name) {}
/** @param {string} name */ function foo(name) {} // thiếu description
/** Mô tả */ function foo() { return 1; } // thiếu @returns

// ✅ ĐÚNG
/**
 * Tính tổng giá trị
 * @param {string} name - Tên người dùng
 * @param {number} age - Tuổi
 * @returns {string} Thông tin người dùng
 */
function getUserInfo(name, age) {
    return `${name} - ${age}`;
}

3.4. Property Documentation (Documentation Cho Property)

Quy tắc: Bắt buộc có @property cho mỗi property trong object.

// ❌ SAI
/** */ const obj = { name: "John" };

// ✅ ĐÚNG
/**
 * @property {string} name - Tên người dùng
 * @property {number} age - Tuổi
 */
const obj = { name: "John", age: 30 };

3.5. Type Validation (Kiểm Tra Type)

Quy tắc: Type trong JSDoc phải hợp lệ.

// ⚠️ CẢNH BÁO
/** @param {Stirng} name */ // sai chính tả String

// ✅ ĐÚNG
/** @param {string} name */

4. Import Rules - Quy Tắc Import Modules

4.1. Import Order (Thứ Tự Import)

Quy tắc: Sắp xếp import theo thứ tự:

  1. Builtin (Node.js modules)
  2. External (npm packages)
  3. Internal (project modules)
  4. Parent (../)
  5. Sibling (./)
  6. Index (./index)

Alphabetize: Sắp xếp theo thứ tự alphabet.

// ❌ SAI
import "./local";
import "fs";
import "lodash";

// ✅ ĐÚNG
import "fs"; // builtin
import "lodash"; // external
import "../utils/helper"; // parent
import "./local"; // sibling

4.2. No Unresolved (Không Import Không Tồn Tại)

Quy tắc: Cấm import module không tồn tại.

// ❌ SAI
import { foo } from "./non-existent";

// ✅ ĐÚNG
import { foo } from "./existing-file";

4.3. No Cycle (Không Circular Dependency)

Quy tắc: Cảnh báo circular dependency (import lẫn nhau).

// ⚠️ CẢNH BÁO
// fileA.js
import { bar } from "./fileB";

// fileB.js
import { foo } from "./fileA"; // circular dependency

// ✅ ĐÚNG: Tách shared code ra module riêng
// shared.js
export function sharedFunction() {}

// fileA.js
import { sharedFunction } from "./shared";

// fileB.js
import { sharedFunction } from "./shared";

5. Security Rules - Quy Tắc Bảo Mật

5.1. Object Injection (Object Injection)

Quy tắc: Cảnh báo object injection vulnerability.

// ⚠️ CẢNH BÁO
obj[userInput]; // user input dùng làm key có thể nguy hiểm

// ✅ ĐÚNG: Validate/sanitize userInput trước khi dùng
const allowedKeys = ["key1", "key2"];
if (allowedKeys.includes(userInput)) {
    return obj[userInput];
}

5.2. File Path (Đường Dẫn File)

Quy tắc: Cảnh báo dùng biến động cho file path.

// ⚠️ CẢNH BÁO
fs.readFile(userInput); // path từ user input có thể nguy hiểm

// ✅ ĐÚNG: Validate/sanitize path, dùng path.join()
const path = require("path");
const sanitizedPath = path.join(baseDir, path.basename(userInput));
fs.readFile(sanitizedPath);

5.3. Eval (Eval)

Quy tắc: Cấm dùng eval() với expression động.

// ❌ SAI (CỰC KỲ NGUY HIỂM)
eval(userInput); // có thể execute code tùy ý

// ✅ ĐÚNG: Dùng JSON.parse() hoặc safe alternatives
const data = JSON.parse(userInput);

5.4. Timing Attacks (Timing Attack)

Quy tắc: Cảnh báo timing attack trong string comparison.

// ⚠️ CẢNH BÁO
if (password === userPassword) {} // có thể bị timing attack

// ✅ ĐÚNG: Dùng crypto.timingSafeEqual() hoặc bcrypt.compare()
const crypto = require("crypto");
if (crypto.timingSafeEqual(Buffer.from(password), Buffer.from(userPassword))) {}

6. Promise Rules - Quy Tắc Xử Lý Promise

6.1. Always Return (Luôn Return)

Quy tắc: Cảnh báo Promise handler không return.

// ⚠️ CẢNH BÁO
promise.then(() => {
    doSomething(); // thiếu return
});

// ✅ ĐÚNG
promise.then(() => {
    return doSomething();
});

6.2. Catch or Return (Catch Hoặc Return)

Quy tắc: Cấm Promise không có catch handler.

// ❌ SAI
promise.then(() => {}); // thiếu catch, lỗi sẽ bị nuốt

// ✅ ĐÚNG
promise.then(() => {}).catch(err => {
    console.error(err);
});

6.3. No Nesting (Không Lồng Nhau)

Quy tắc: Cảnh báo Promise lồng nhau quá sâu.

// ⚠️ CẢNH BÁO
promise.then(() => {
    promise2.then(() => {
        promise3.then(() => {}); // lồng nhau quá sâu
    });
});

// ✅ ĐÚNG: Dùng async/await hoặc Promise chain phẳng
const result1 = await promise;
const result2 = await promise2;
const result3 = await promise3;

6.4. Prefer Await (Ưu Tiên Await)

Quy tắc: Khuyến khích dùng await thay .then().

// ⚠️ CẢNH BÁO
promise.then(result => {
    console.log(result);
});

// ✅ ĐÚNG
const result = await promise;
console.log(result);

6.5. Prefer Await to Callbacks (Ưu Tiên Await Thay Callback)

Quy tắc: Khuyến khích dùng async/await thay callback.

// ⚠️ CẢNH BÁO
fs.readFile(path, (err, data) => {
    if (err) { throw err; }
    console.log(data);
});

// ✅ ĐÚNG
const data = await fs.promises.readFile(path);
console.log(data);

7. Naming Conventions - Quy Tắc Đặt Tên

7.1. Variables & Functions (Biến & Hàm)

Quy tắc: Sử dụng camelCase.

// ✅ ĐÚNG
const userName = "John";
const totalPrice = 1000;
function calculateTotal() {}
function getUserInfo() {}

7.2. Constants (Hằng Số)

Quy tắc: Sử dụng UPPER_SNAKE_CASE.

// ✅ ĐÚNG
const API_BASE_URL = "https://api.example.com";
const MAX_RETRY_COUNT = 3;

7.3. Classes (Lớp)

Quy tắc: Sử dụng PascalCase.

// ✅ ĐÚNG
class UserService {}
class OrderController {}

7.4. Private Members (Thành Viên Riêng Tư)

Quy tắc: Prefix _ cho private members.

// ✅ ĐÚNG
class User {
    constructor(name) {
        this._name = name; // private
    }

    getName() {
        return this._name;
    }
}

7.5. Boolean (Boolean)

Quy tắc: Prefix is, has, should, can.

// ✅ ĐÚNG
const isLoading = true;
const hasPermission = false;
const shouldRetry = true;
const canEdit = false;

8. File Structure - Cấu Trúc File

8.1. File Organization (Tổ Chức File)

Quy tắc: Sắp xếp code theo thứ tự:

  1. Imports (external → internal)
  2. Constants
  3. Types/Interfaces (nếu có)
  4. Functions/Classes
  5. Exports
// ✅ ĐÚNG
// 1. Imports
import "fs";
import "lodash";
import "../utils/helper";

// 2. Constants
const API_BASE_URL = "https://api.example.com";

// 3. Functions
/**
 * Tính tổng giá trị
 * @param {number[]} numbers - Danh sách số
 * @returns {number} Tổng giá trị
 */
function calculateSum(numbers) {
    return numbers.reduce((sum, num) => sum + num, 0);
}

// 4. Exports
export default calculateSum;

8.2. JSON Files (File JSON)

Quy tắc: JSON files dùng indent 2 spaces (chuẩn JSON).

{
  "name": "project",
  "version": "1.0.0",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}

8.3. Test Files (File Test)

Quy tắc: Test files có rules riêng, lỏng hơn:

  • Không bắt buộc JSDoc cho test functions
  • Cho phép callback lồng nhau nhiều
  • Cho phép dynamic file path
// ✅ ĐÚNG: Test file không cần JSDoc
describe("calculateSum", () => {
    it("should return sum of numbers", () => {
        expect(calculateSum([1, 2, 3])).toBe(6);
    });
});

📝 Checklist Trước Khi Commit

Trước khi commit code, đảm bảo:

  • Code pass ESLint (không có lỗi)
  • Code pass Prettier (format đúng)
  • Có JSDoc cho tất cả functions/classes/methods
  • Không có unused variables (hoặc prefix _)
  • Không có var, chỉ dùng let/const
  • Dùng === thay ==
  • Dùng const khi có thể
  • Có error handling cho Promise
  • Không có security vulnerabilities
  • Import được sắp xếp đúng thứ tự
  • File có newline ở cuối
